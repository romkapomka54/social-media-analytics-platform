# План Проекту: Social Media Analytics Platform

## 1. Загальний опис проекту
Створити веб-додаток/платформу, яка дозволить користувачам (організаціям) аналізувати коментарі під відео на YouTube, TikTok, Instagram, а також транскрибувати відео з цих платформ. Основна мета — моніторинг громадської думки, аналіз поведінки коментарів (позитивні/негативні) та формування загального уявлення про реакцію аудиторії на контент.

## 2. Ключові функції додатку

### 2.1. Аналіз коментарів
- [ ] Збір коментарів з YouTube, TikTok, Instagram.
- [ ] Аналіз тональності (сентименту): Визначення, чи є коментар позитивним, негативним або нейтральним.
- [ ] Аналіз ключових тем/ключових слів: Виявлення найчастіше обговорюваних тем у коментарях.
- [ ] Визначення загальної думки: Формування узагальненого висновку про ставлення аудиторії до відео/контенту.
- [ ] Ідентифікація трендів: Відстеження змін у тональності та темах з часом.
- [ ] Відображення статистики та візуалізацій.

### 2.2. Транскрибація відео
- [ ] Можливість транскрибувати аудіодоріжки з відео (TikTok, Instagram, YouTube).
- [ ] Розумна економія ресурсів: Автоматичне витягування існуючих субтитрів (якщо вони є) перед запуском транскрибації для економії токенів API.
- [ ] Конвертація мови в текст.
- [ ] Спроба ідентифікації різних голосів у транскрипції для форматування діалогів та прив'язки до "тредів повідомлень".

### 2.3. Тенантна система (для організацій)
- [ ] Реєстрація організацій: Кожна організація має свій ізольований простір.
- [ ] Керування користувачами: Додавання та видалення користувачів всередині організації.
- [ ] Ролі та права доступу: Призначення різних ролей користувачам (наприклад, адміністратор, аналітик, переглядач) з відповідними правами доступу до функцій та даних.
- [ ] Керування каналами/джерелами даних: Організації можуть додавати свої власні канали YouTube, профілі TikTok, акаунти Instagram для аналізу.

### 2.4. Автоматизований відділ турботи (Approval Workflow / Human-in-the-Loop)
- [ ] Виявлення (Polling): ШІ-агент (OpenClaw) автоматично відстежує нові коментарі через API платформ.
- [ ] Генерація чернетки (Drafting): LLM (NVIDIA GLM-5) аналізує контекст (наприклад, скаргу від "детрактора") і формує професійну, персоналізовану відповідь від імені компанії для згладжування конфлікту чи подяки.
- [ ] Збереження: Чернетка зберігається у базі даних (Supabase) зі статусом "Очікує підтвердження" без прямої публікації в інтернет.
- [ ] Схвалення (Approval): Менеджер компанії заходить у веб-дашборд, бачить список нових коментарів із готовими ШІ-відповідями. Він може відредагувати текст або просто натиснути кнопку "Схвалити".
- [ ] Авто-публікація: Після схвалення людиною, агент автоматично відправляє відповідь під відповідне відео/пост у соцмережі.

## 5. Архітектура додатку та Мови Програмування

### 5.1. Архітектура: Мікросервіси, ШІ-Агенти та Бессерверні функції
Для такого складного проекту ідеальний вибір — **мікросервісна архітектура**, доповнена **бессерверними функціями** та **оркестрацією автономних агентів**.

*   **Мікросервісна архітектура (Microservices):** Розбиває додаток на невеликі, незалежні сервіси, що покращує масштабованість, гнучкість розгортання та підтримку коду.
    *   **Приклади мікросервісів:** Сервіс збору даних (Data Ingestion), Сервіс обробки та аналізу даних (Sentiment Analysis, Topic Modeling на базі NVIDIA GLM-5), Сервіс транскрибації, Сервіс візуалізації та звітів.

*   **Оркестрація Агентів (OpenClaw):** Використовується як автономний фоновий процес для моніторингу (polling), звернення до API соцмереж, генерації draft-відповідей та їх автоматичної публікації після схвалення людиною.

*   **Бессерверні функції (Serverless Functions) / Headless Functions:** Чудове доповнення для подієво-орієнтованих завдань, виконання за розкладом або трансформації даних. Економічно ефективні та високомасштабовані.
    *   **Використання:** Обробка вхідних даних у реальному часі, планові завдання, трансформації даних.

### 5.2. Мови Програмування: Python та JavaScript
Ці мови ідеально доповнюють одна одну:

*   **Python:** Основний для **бекенду та аналітики**, особливо для **збору та обробки даних** (аналіз тональності, тематичне моделювання, виявлення тенденцій). Сила в багатій екосистемі бібліотек для Data Science, машинного навчання та обробки природної мови (NLP) (pandas, scikit-learn, NLTK) та фреймворках (Flask, FastAPI).
    *   Твої знання Python значно прискорять розробку аналітичних сервісів.

*   **JavaScript (Frontend та Node.js для Backend):**
    *   **Фронтенд:** Критично важливий для розробки **клієнтської частини (UI/UX)**. Фреймворки (React, Angular, Vue.js) для динамічного та інтерактивного інтерфейсу.
    *   **Node.js (Backend):** Може використовуватися для деяких бекенд-мікросервісів, що вимагають високої продуктивності вводу/виводу або взаємодії в реальному часі (API-шлюзи).
    *   Твої плани вивчати JavaScript ідеально вписуються в потреби проекту.

### 5.3. Складові додатку (крім автентифікації):

*   **Безпека та шифрування даних:** **Критично важливо!** Усі чутливі дані мають бути зашифровані (як під час передачі, так і під час зберігання). Реалізація детальних механізмів контролю доступу (RBAC). Постійний моніторинг безпеки та ведення журналів аудиту. Архітектура повинна враховувати це майбутнє розширення.

*   **Тенантна архітектура:** Кожна організація має свій ізольований простір та дані (наприклад, через `organization_id` у базі даних). Управління користувачами: реєстрація, профілі, ролі та права доступу.

## 9. Важливі Нотатки та Обговорення

### 9.1. Вимоги до Бази Даних
*   Вибір: PostgreSQL або Supabase. **Supabase — дуже ефективне рішення.**
*   Зберігання: Фокус на зберіганні **тредів коментарів** (батьківський коментар + відповіді).
*   Об'єм та вартість: Потребує детальної оцінки після визначення очікуваного навантаження (кількість організацій, користувачів, обсяг коментарів/відео). На початковому етапі слід розраховувати на гнучкі тарифи хмарних провайдерів.

### 9.2. API Соціальних Мереж
*   Детальний аналіз документації API YouTube, TikTok, Instagram.
*   Визначення необхідних API-запитів, їхніх обмежень та методів аутентифікації.

### 9.3. Безпека
*   Шифрування даних (at rest та in transit).
*   Контроль доступу (RBAC).
*   Аудит та моніторинг.
*   Сертифікація Google в майбутньому.

### 9.4. GitHub Співпраця
*   Пряма робота в репозиторії `social-media-analytics-platform` (для мене) з використанням токена.
*   Для інших репозиторіїв — форки або Pull Requests.
