# 8. Внутрішній API та Інтеграції (API & Integrations Docs)

Цей модуль описує структуру нашого внутрішнього REST API для роботи веб-дашборду, керування ШІ-моделями та фоновими процесами, а також взаємодію з зовнішніми сервісами.

## 8.1. API керування ШІ (AI Configuration API)

Ендпоінти для гнучкого налаштування мульти-LLM системи для кожного клієнта (тенанта).

| Метод | Ендпоінт | Опис |
|-------|----------|------|
| `GET` | `/api/ai/providers` | Отримання списку доступних провайдерів (NVIDIA, Anthropic) з таблиці `ai_providers` |
| `GET` | `/api/ai/models` | Отримання списку моделей, які підтримують необхідні функції (наприклад, `supports_tools: true`) |
| `POST` | `/api/tenant/ai-configs` | Збереження або оновлення унікального системного промпту та вибору моделі для конкретного клієнта (`tenant_id`) та конкретної задачі (`use case`) в таблиці `tenant_ai_configs` |

## 8.2. API фонової обробки (Cron & Dispatching)

Внутрішні ендпоінти для управління процесом збору коментарів (Polling).

| Метод | Ендпоінт | Опис |
|-------|----------|------|
| `GET` | `/api/cron/youtube/sync` | **Диспетчер (Dispatcher).** Запускається за розкладом, збирає список підключених каналів і відправляє команди Робітникам |
| `POST` | `/api/cron/youtube/sync/[tenantId]` | **Робітник (Worker).** Приймає запит від Диспетчера, миттєво повертає `202 Accepted` і починає асинхронно стягувати коментарі та аналізувати їх через LangChain |

## 8.3. API Автоматизованого відділу турботи (Approval Workflow)

Ендпоінти, з якими безпосередньо взаємодіє клієнтський фронтенд для перевірки та публікації відповідей (Smart Inbox).

| Метод | Ендпоінт | Опис |
|-------|----------|------|
| `POST` | `/api/comments/analyze` | Внутрішній виклик методу LangChain `.chatWithTools()`, який аналізує тональність зібраних коментарів і генерує чернетку відповіді, зберігаючи її в БД |
| `GET` | `/api/drafts/pending` | Отримання фронтендом списку згенерованих ШІ чернеток відповідей, які очікують на схвалення людиною (менеджером) |
| `POST` | `/api/drafts/approve` | Підтвердження чернетки менеджером. Цей ендпоінт ініціює POST-запит до офіційного API відповідної соцмережі для публікації коментаря |

## 8.4. Зовнішні інтеграції (Third-Party APIs)

Усі звернення до зовнішніх платформ здійснюються через серверні функції Python, з використанням збережених OAuth-токенів клієнтів (таблиця `oauth_tokens`).

- **Meta Graph API (Instagram):**
  - Отримання коментарів: `GET /{ig-media-id}/comments`
  - Публікація відповідей: `POST /{ig-comment-id}/replies`
- **YouTube Data API v3:**
  - Збір треду коментарів: `GET /commentThreads/list`
  - Публікація відповідей: `POST /comments/insert`
  - Витягування субтитрів: `GET /youtube/v3/captions/{captionId}`
- **TikTok API:** (Обмежений доступ). Моніторинг через Webhooks для топ-коментарів, або використання кастомних скраперів у Python-робітниках як резервний механізм.
