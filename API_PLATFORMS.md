## 2. Автентифікація та Доступ
Це найголовніший розділ, який нам потрібен. Для Instagram, наприклад, є два шляхи:

### 2.1. Бізнес-логін (Business Login)
*   **Як це працює?** Користувач дає дозвіл, ми отримуємо токен.
*   **Термін дії токена?** (60 днів для довготривалого токена).
*   **Необхідні права (permissions/scopes)?** (напр., `instagram_basic`, `instagram_manage_insights`).

### 2.2. Логін через Facebook (Facebook Login for Business)
*   **Як це працює?** Складніший шлях, для роботи з багатьма клієнтами через Business Manager.
*   **Створення App в Meta for Developers.**
*   **Системні користувачі (System Users) та їхні токени.**
*   **Зв'язок з Facebook Page.**

## 3. Основні Endpoint'и (Точки доступу)
Тут ми описуємо, які дані взагалі можна отримати. Наприклад, для Instagram це:

*   `GET /me/media` – список медіа.
*   `GET /{media-id}/insights` – аналітика для конкретного поста (лайки, коментарі, охоплення).
*   `GET /{media-id}/comments` – список коментарів.
*   `POST /{media-id}/comments` – відповідь на коментар (якщо є права).

## 4. Ліміти (Rate Limiting)
Це критично важливо для нашої архітектури!

*   **Приклад Instagram:** 200 запитів на годину на одного користувача.
*   **Як це працює?** Це не "200 на весь додаток", а 200 на кожен Instagram-акаунт, дані якого ми читаємо.
*   **Моніторинг:** Як відстежувати ліміт (відповідь API містить спеціальні заголовки `X-Business-Use-Case-Usage`).

## 5. Зміни, що плануються (Deprecations)
Аналіз документації Meta показує, що відбуваються масштабні зміни, про які ми мусимо знати, щоб не писати код, який перестане працювати завтра:

### 5.1. Meta (Facebook/Instagram)
*   **Березень 2026:** Оновлення mTLS сертифікатів для Webhook. Якщо ми будемо використовувати вебхуки, нам доведеться оновити довірені сертифікати на сервері до 31 березня, інакше ми перестанемо отримувати події.
*   **Червень 2026:** Повне зникнення старих метрик (охоплення, покази) та перехід на нові метрики "Viewers" та "Views". Це означає, що наші аналітичні звіти доведеться повністю переробляти під нові метрики вже через 3 місяці.
*   **Січень 2026 (вже минув!):** Deadline для міграції Marketing API на нові об'єкти (IG User, IG Media). Добре, що ми не спізнилися з плануванням.
