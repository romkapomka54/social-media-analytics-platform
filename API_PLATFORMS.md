# API Платформи для Соціальних Мереж

Цей документ містить технічний аналіз API різних соціальних платформ для нашого проекту. Він слугує основою для планування розробки та інтеграції.

## Як читати цей документ

Для кожної платформи ми заповнюємо однакову структуру, щоб мати змогу порівнювати їхні можливості та вимоги.

---

## 1. Instagram & Facebook (Meta)

### 1.1. Загальна Інформація
*   Офіційна назва API: Instagram Graph API, Facebook Graph API
*   Поточна версія: v25.0 (актуальна з 18 лютого 2026)
*   Дата останніх змін: Лютий 2026
*   Графік змін (Changelog): [Посилання на офіційний Changelog Meta for Developers]

### 1.2. Автентифікація та Доступ
*   Метод: OAuth 2.0
*   Типи доступу:
    *   *Бізнес-логін:* Для доступу до одного акаунту. Термін дії токена — 60 днів.
    *   *Логін через Facebook:* Для роботи з клієнтами через Business Manager. Вимагає створення App та системних користувачів.
*   Необхідні права (permissions): instagram_basic, instagram_manage_insights, pages_read_engagement (для читання коментарів) тощо.

### 1.3. Основі Endpoint'и (Приклади)
*   Медіа: GET /me/media – список постів.
*   Аналітика: GET /{media-id}/insights – аналітика для конкретного поста (лайки, коментарі, охоплення).
*   Коментарі: GET /{media-id}/comments – список коментарів.
*   Відповіді: POST /{comment-id}/replies – відповісти на коментар (якщо є права).

### 1.4. Ліміти (Rate Limiting)
*   Основний ліміт: 200 запитів на годину на одного користувача.
*   Моніторинг: Використовувати заголовки X-Business-Use-Case-Usage у відповідях API.

### 1.5. Майбутні зміни (Важливо!)
*   До 31 березня 2026: Оновити mTLS сертифікати для Webhook.
*   З червня 2026: Старі метрики (Reach, Impressions) зникнуть. Перехід на метрики "Viewers" та "Views". Потрібно буде переробити аналітичні звіти.
*   Січень 2026: Міграція Marketing API на нові об'єкти (`IG User`, `IG Media`) завершена. Ми не спізнилися.

## 2. TikTok

### 2.1. Загальна Інформація
*   Офіційна назва API: TikTok Business APIs (або TikTok Display API для базового доступу)
*   Поточна версія: v2 (актуальна для бізнес-API)
*   Типи API: Існує два основних типи доступу:
    *   *Standard TikTok Connection* (споживчий API): Базова інтеграція, постінг, 4 основні метрики (лайки, коментарі, шери, перегляди).
    *   *TikTok Business Connection* (маркетинговий API): Розширена аналітика, кастомні обкладинки, демографія аудиторії.
*   Графік змін (Changelog): [Посилання на офіційний TikTok Developer Portal]

### 2.2. Автентифікація та Доступ
*   Метод: OAuth 2.0
*   Процес: 
    1.  Реєстрація додатку в [TikTok Developer Portal](https://developers.tiktok.com/)
    2.  Отримання client_key (App ID) та client_secret
    3.  Запит авторизації через https://www.tiktok.com/v2/auth/authorize
    4.  Обмін authorization_code на access_token
*   Термін дії токена: Access token діє 24 години, refresh token може використовуватись для оновлення
*   Необхідні права (scopes):
    *   user.info.basic - базова інформація користувача
    *   video.list - доступ до списку відео
    *   video.upload - завантаження відео (для публікації)
    *   video.insights - розширена аналітика (для Business API)
    *   user.insights - демографічні дані аудиторії

### 2.3. Основні Endpoint'и
*   Інформація про користувача: GET /v2/user/info/ (open_id, union_id, display_name, avatar_url)
*   Список відео: GET /v2/video/list/ (отримати список відео користувача)
*   Деталі відео: GET /v2/video/query/ (детальна інформація про конкретне відео)
*   Коментарі: *(Потребує додаткового дослідження - точний endpoint не знайдено в документації)*
*   Завантаження відео: POST /v2/video/upload/ (публікація відео)

### 2.4. Метрики та Аналітика
| Метрика | Доступ | Опис |
|---------|--------|------|
| like_count | ✅ Standard | Кількість вподобань |
| comment_count | ✅ Standard | Кількість коментарів |
| share_count | ✅ Standard | Кількість поширень |
| view_count | ✅ Standard | Кількість переглядів |
| total_time_watched | ✅ Business | Загальний час перегляду |
| average_time_watched | ✅ Business | Середній час перегляду |
| video_view_retention | ✅ Business | Графік утримання аудиторії |
| Демографія (gender/country) | ✅ Business | Аудиторія за статтю/країною |

### 2.5. Ліміти (Rate Limits)
*   Основний ліміт: Залежить від endpoint'а; для базового доступу ~100-200 запитів на день.
*   Публікація: Строгі обмеження для запобігання спаму.
*   Стратегія: Використовувати експоненційну затримку (exponential backoff) при помилках 429/5xx.

### 2.6. Майбутні зміни та важливі нотатки
*   Webhooks: TikTok підтримує вебхуки для реального часу, але повний каталог доступний після логіну в Partner Center.
*   Коментарі: Документація щодо отримання коментарів обмежена. Можливо, знадобиться використовувати парсинг або моніторинг через Mentions API.
*   Hashtag Listening: Для відстеження хештегів потрібен Business API та налаштування в TikTok Hashtags Manager.
*   Пісочниця (Sandbox): Доступна для тестування через "Generate test access token".
